# Phishing Email Detector

## Project Overview

Live demo: https://phishguarduk.streamlit.app

A machine-learning prototype that classifies emails as **phishing** or **legitimate** and
explains the prediction in plain English.  Built as an educational tool to demonstrate
how text classification and explainable AI techniques (LIME, SHAP) can support
cybersecurity awareness training.

The system uses a Logistic Regression classifier trained on TF-IDF features.
All design decisions — model choice, vectoriser configuration, and explanation
method — follow the project plan described in the PPRS document.

## How to Run

Run all commands from the **project root** directory.

### Step 1 — Place your dataset

Put your labelled email CSV at:

```
data/raw/dataset.csv
```

The file must contain two columns:

| Column          | Description                                               |
|-----------------|-----------------------------------------------------------|
| `text_combined` | Full email text (subject + body concatenated)             |
| `label`         | `0` = legitimate, `1` = phishing (or `legitimate`/`phishing`) |

### Step 2 — Preprocess

```bash
python src/preprocess.py
```

Cleans the text (URL/email tokenisation, lowercasing), encodes labels, saves
`data/processed/processed.csv`, and writes dataset statistics to
`outputs/evaluation/dataset_stats.json`.

### Step 3 — Train

```bash
python src/train.py
```

Trains four models (Logistic Regression, Random Forest, Naive Bayes, XGBoost) on the
same train/test split, evaluates all three, and saves:

- `models/model_logreg.joblib` (primary deployed model)
- `models/model_rf.joblib`
- `models/model_nb.joblib`
- `models/tfidf_vectorizer.joblib`
- `outputs/evaluation/` — classification report, confusion matrix, ROC curve,
  cross-validation results, and model comparison table/chart.

### Step 4 — Launch the app

```bash
streamlit run src/app_streamlit.py
```

## Model Performance

After running `train.py`, full results are saved to `outputs/evaluation/`:

| File                          | Contents                                      |
|-------------------------------|-----------------------------------------------|
| `classification_report.txt`   | Per-class precision, recall, F1, accuracy     |
| `classification_report.json`  | Machine-readable version of the above         |
| `confusion_matrix.png`        | Confusion matrix heatmap                      |
| `roc_curve.png`               | ROC curve with AUC score                      |
| `cross_validation_results.txt`| 5-fold CV mean ± std accuracy                 |
| `model_comparison.csv`        | Accuracy / precision / recall / F1 / time     |
| `model_comparison.png`        | Grouped bar chart of the above                |
| `dataset_stats.json`          | Sample counts and class distribution          |

The app's sidebar loads and displays these files automatically.

## How to Interpret Results

**Confidence score**
The model's predicted probability for the chosen class.  A score of 0.95 means
95% confidence.  Scores near 0.5 indicate uncertainty.

**LIME / SHAP bar chart**
Each bar represents a word.  Red bars push the prediction toward *phishing*;
green bars push toward *legitimate*.  Longer bars mean stronger influence.

**Learning mode**
Enable before clicking "Analyse email".  Guess the label first, then see the
model's prediction, your running score, and an educational tip about the key
features.

## Limitations

- **Single dataset:** The model was trained on one combined dataset.
  Cross-dataset evaluation was planned but not completed within the project
  timeline — this is documented as future work.
- **No transformer model:** DistilBERT or a fine-tuned BERT model was
  identified as potential future work but was out of scope for this prototype.
- **Text only:** The system analyses text content only.  URL reputation
  checking, attachment scanning, and email header inspection are not
  implemented.
- **Concept drift:** Phishing language evolves rapidly.  The model should
  be retrained periodically on fresh data.
